# Sesja 2026-05-20 — Android Port + Ikona

## Cel
Pełny port SavvyCAN na Androida z zachowaniem wszystkich funkcji desktopowych.
Zmiana ikony aplikacji na oryginalne logo SavvyCAN.

---

## Zrealizowane

### 1. Port C++ — wszystkie moduły desktopowe
- **Core**: `can_structs`, `utility`, `simplecrypt`, `futuristic_theme`
- **Modele**: `framestore`, `canframemodel`, `canfilter`, `filterutility`
- **I/O**: `framefileio` (12 formatów), `blfhandler`, `pcaplite`
- **DBC**: `dbc_classes`, `dbchandler`
- **Protokoły**: `isotp_handler` (ISO 15765-2), `j1939_handler` (J1939), `uds_handler` (UDS)

### 2. Mocki dla niedostępnych modułów Qt
Zastąpione w `android/mock_serialbus/`:
- `QCanBusFrame`, `QCanBusDevice`, `QCanBus` (QtSerialBus)
- `QSerialPort`, `QSerialPortInfo` (QtSerialPort)
- `QtHttpServer`, `QWebSocket`, `QWebSocketServer` (stuby)
- `pybind11`, `PythonBridge`, `PythonConsole` (stuby)
- `MainWindow`, `CANConManager` (stuby dla zależności z `bus_protocols/`)

### 3. Android-specific
- **BluetoothManager** — pełna implementacja JNI (BluetoothAdapter, RFCOMM/SPP)
- **WifiManager** — TCP + parsowanie ramek SLCAN
- **UsbHostManager** — USB OTG z JNI (PEAK PCAN-USB)
- **FrameListModel** — QAbstractListModel eksponowany do QML

### 4. Interfejs QML (15 stron)
1. **Connection** — Bluetooth, WiFi, USB OTG
2. **Frame List** — filtrowanie, overwrite, quick send
3. **File I/O** — 12 formatów
4. **Graph View** — placeholder pod Qt Charts
5. **Sniffer** — wyzwalanie po ID i wzorcu
6. **Signal Viewer** — dekodowanie sygnałów DBC
7. **UDS Scanner** — skaner diagnostyczny
8. **ISOTP Interpreter** — dekoder ISO 15765-2
9. **Fuzzing** — fuzzer CAN
10. **Frame Sender** — wysyłanie ramek
11. **Playback** — odtwarzanie ramek
12. **CAN Bridge** — mostkowanie magistrali
13. **DBC Editor** — edytor baz DBC
14. **File Compare** — porównywarka logów
15. **Settings** — konfiguracja + About

### 5. Ikona aplikacji
- Pobrano `icons/icon.png` (1084×1104 RGBA)
- Przeskalowano do mipmap-mdpi/hdpi/xhdpi/xxhdpi/xxxhdpi (48-192px)
- Ustawiono `android:icon="@mipmap/icon"` w manifeście
- Poprawiono theme.xml na `Theme.DeviceDefault.NoActionBar`

### 6. Budowanie
```bash
cd SavvyCAN
cmake -B build_android -S android \
  -DCMAKE_TOOLCHAIN_FILE=$ANDROID_NDK/build/cmake/android.toolchain.cmake \
  -DANDROID_ABI=arm64-v8a -DANDROID_PLATFORM=android-26 \
  -DANDROID_SDK_ROOT=$ANDROID_SDK \
  -DCMAKE_BUILD_TYPE=Release -GNinja \
  -DQt6_DIR=$QT_ANDROID/lib/cmake/Qt6 \
  -DQT_HOST_PATH=$QT_HOST \
  -DCMAKE_FIND_ROOT_PATH_MODE_PACKAGE=BOTH
JAVA_HOME=/usr/lib/jvm/java-21-openjdk-amd64 cmake --build build_android --target apk
```

Wynik: **SavvyCAN-Android.apk** (65 MB, arm64-v8a, minSdk 26)

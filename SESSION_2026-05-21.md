# SavvyCAN Desktop Build + CAN Testing — Sesja 2026-05-21

## Gałąź: `master` (po przełączeniu z `android-port`)

### 1. Desktop Linux Build — SavvyCAN skompilowany

**Binarka**: `build_desktop/SavvyCAN` — 7.8 MB ELF 64-bit
**Uruchamia się** poprawnie (locale: pl_PL/en)

**Problemy rozwiązane przy kompilacji**:

| Problem | Rozwiązanie |
|---------|-------------|
| Qt6SerialBus/SerialPort brak w custom Qt 6.10.2 gcc_64 | Użyto systemowych pakietów (`qt6-serialbus-dev`, `qt6-serialport-dev`) przez `CMAKE_PREFIX_PATH=/usr/lib/x86_64-linux-gnu/cmake` |
| Qt6HttpServer, Qt6WebSockets — brak w systemie | Stuby w `stubs/` (QHttpServer, QWebSocketServer, QWebSocket, QTcpServer) |
| REST API (`restapiserver.cpp`) nie kompiluje się z stubami | Wyłączono `#include "restapiserver.h"` i kod `RestApiServer` w `mainwindow.h/cpp` |
| `QCanBusFrame` / `QSerialPort` nagłówki nie znajdowane | Dodano ścieżki include systemowe `/usr/include/x86_64-linux-gnu/qt6/QtSerialBus` i `QtSerialPort` |
| Warning `-Werror=deprecated` dla `QImage::mirrored` | Dodano `-Wno-error=deprecated-declarations` |

### 2. CAN Communication — testy magistrali

**Hardware**: PEAK PCAN-USB (can0) + ESP32 MCP2515 250kbps 8MHz

**Testy**:
- PCAN → CAN → ESP32 MCP2515: ACK działa
- PCAN → CAN → ESP32 → BLE notify: 5/5 ramek ✅
- BLE → ESP32 (CAN TX char) → MCP2515 → CAN → PCAN: 4/4 ramek ✅
- Wcześniejszy problem (CAN RX 0 ramek): brak terminacji 120Ω na magistrali — PCAN dostarczył terminację

### 3. ESP32 Firmware
- `esp32_firmware/ble_can_bridge/ble_can_bridge.ino` — wgrany
- CCCD descriptor (0x2902) dodany do CAN RX characteristic
- Działa dwukierunkowo: BLE ↔ CAN

### 6. Budowanie desktop (na przyszłość)
```bash
cd SavvyCAN
cmake -B build_desktop -S . \
  -DCMAKE_PREFIX_PATH=/usr/lib/x86_64-linux-gnu/cmake \
  -DCMAKE_BUILD_TYPE=Release -GNinja -DBUILD_TESTS=OFF
cmake --build build_desktop -j$(nproc)
```
Wynik: `build_desktop/SavvyCAN`
